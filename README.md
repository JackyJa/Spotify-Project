# 🎵 Spotify Music Management System (C++ / Qt)

یک پروژه مدیریت موسیقی شبیه‌ساز اسپاتیفای، توسعه‌یافته با **C++** و فریم‌ورک **Qt6**. این پروژه با رعایت اصول شی‌گرایی، معماری MVC، الگوی طراحی Repository، پایگاه داده SQLite و ادغام یک دستیار هوش مصنوعی (RAG) توسعه یافته است.

---

## 🎓 اطلاعات دانشگاه
- **دانشگاه:** دانشگاه اصفهان
- **درس:** برنامه‌نویسی پیشرفته
- **استاد درس:** دکتر احسان مهدوی
- **ترم:** تابستان ۱۴۰۵
- **طراحان پروژه:** حسین مومنی، مهرشاد جعفری، آتوسا مسعودی نیا، عرفان حبیب، امیرمحسن قاسمی

---

## ✨ ویژگی‌های کلیدی (Features)

### 👥 حساب کاربری (Authentication & Accounts)
- سیستم ثبت‌نام و ورود با مدیریت خطا.
- پشتیبانی از دو نقش: **هنرمند (Artist)** و **شنونده (Listener)**.
- بررسی قدرت رمز عبور با **Regex** (ضعیف، متوسط، قوی) و جلوگیری از ثبت‌نام با رمز ضعیف.
- جلوگیری از ثبت‌نام با Username تکراری.
- آپلود عکس پروفایل و نمایش آن با گوشه‌های گرد (Rounded Corners) در داشبورد.
- قابلیت ویرایش اطلاعات (نام، بیوگرافی، رمز عبور) و حذف اکانت.

### 🎤 پنل هنرمند (Artist Dashboard)
- **مدیریت آلبوم (CRUD):** ایجاد، ویرایش و حذف آلبوم همراه با آپلود عکس کاور. (حذف آلبوم باعث حذف تمام آهنگ‌های داخل آن می‌شود).
- **مدیریت آهنگ (CRUD):** افزودن، ویرایش و حذف آهنگ به آلبوم‌ها یا بخش Singles، شامل سال انتشار، سبک (Genre)، فایل صوتی و کاور اختصاصی آهنگ. (حذف آهنگ باعث حذف آن از پلی‌لیست‌های شنوندگان می‌شود).
- نمایش لیست آلبوم‌ها به صورت گرافیکی با تصاویر کوچک (Thumbnails).

### 🎧 پنل شنونده (Listener Dashboard)
- **مدیریت پلی‌لیست (CRUD):** ایجاد، ویرایش محتوا و حذف کامل پلی‌لیست‌ها.
- **کاوش موسیقی:** مشاهده لیست هنرمندان، آلبوم‌ها و افزودن آهنگ‌ها به پلی‌لیست شخصی.
- **لیست علاقه‌مندی‌ها (Favorite Songs):** لایک کردن آهنگ‌ها، نمایش آن‌ها در یک پلی‌لیست اختصاصی و قابلیت Unlike کردن.
- **جستجو و فیلتر:** جستجوی آهنگ‌ها بر اساس نام، سبک و سال انتشار.
- **پخش موسیقی:** پخش فایل‌های صوتی (mp3/wav) با استفاده از `Qt Multimedia`.

### 🤖 دستیار هوش مصنوعی (AI Chatbot - RAG)
- ادغام یک مدل زبانی محلی (LLM - Qwen 2.5) با استفاده از **Python FastAPI**.
- سیستم **RAG (Retrieval-Augmented Generation)**: چت‌بات به دیتابیس برنامه (SQLite) متصل است و به سوالاتی مانند "خواننده آهنگ X کیست؟" بر اساس داده‌های واقعی برنامه پاسخ می‌دهد.

## 🏗️ معماری و مفاهیم شی‌گرایی (Architecture & OOP)

این پروژه با رعایت دقیق اصول مهندسی نرم‌افزار توسعه یافته است:

- **الگوی طراحی MVC (Model-View-Controller):**
  - **Model:** کلاس‌های موجودیت (`Account`, `Song`, `Album`, `Playlist`) و لایه دسترسی به داده.
  - **View:** رابط کاربری توسعه‌یافته با Qt Designer (فایل‌های `.ui`) با تم تاریک اسپاتیفای، افکت‌های شیشه‌ای (Glassmorphism) و هاله نئونی متحرک.
  - **Controller:** کلاس‌های پنجره‌ها که ارتباط بین UI و Database را مدیریت می‌کنند.

- **الگوی Repository:**
  - استفاده از کلاس قالب (Template) به نام `<AbstractRepository<T` برای کپسوله‌سازی.
  - پیاده‌سازی ارث‌بری پیچیده: `ArtistRepository` و `ListenerRepository` از `AccountRepository` ارث بری می‌کنند.

- **پایگاه داده (SQLite):**
  - ذخیره‌سازی دائمی اطلاعات با ماژول `Qt6::Sql`.
  - استفاده از کوئری‌های `JOIN` برای مدیریت روابط پیچیده (مثل آهنگ‌های داخل پلی‌لیست).
  - حفظ یکپارچگی داده‌ها (Data Integrity) هنگام حذف آهنگ یا آلبوم.

## 🚀 نحوه اجرای پروژه (Setup & Installation)

### بخش اول: برنامه اصلی (C++ / Qt)
1. کلون کردن مخزن:
   ```bash
   git clone https://github.com/JackyJa/Spotify-Project.git
   cd Spotify-Project
   ```
2. باز کردن پروژه در **Qt Creator**.
3. اجرای دستور CMake برای دانلود وابستگی‌ها (Sql, Multimedia, Network).
4. بیلد و اجرای برنامه (دیتابیس `spotify.db` به صورت خودکار در اولین اجرا ساخته می‌شود).

### بخش دوم: سرور هوش مصنوعی (Python) - اختیاری
برای استفاده از چت‌بات هوش مصنوعی، باید سرور پایتون را به صورت محلی اجرا کنید:
1. نصب پیش‌نیازها:
   ```bash
   pip install fastapi uvicorn langchain langchain-community transformers torch
   ```
2. اجرای سرور در ترمینال:
   ```bash
   python spot.py
   ```
   (سرور روی پورت `http://127.0.0.1:8000` اجرا می‌شود و برنامه Qt به صورت خودکار به آن متصل می‌شود).
